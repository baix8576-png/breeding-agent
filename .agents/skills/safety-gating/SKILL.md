name: safety-gating
description: 面向 safety_fuse 的安全门禁技能，用于识别高风险操作、定义执行前检查、建立熔断规则、人工审批要求和回滚建议。

## 触发条件
- 新增上云字段、日志输出字段或外部检索能力。
- 新增执行能力可能引入高风险操作。
- 需要定义或更新熔断触发条件与恢复策略。
- 涉及文件删除、覆盖写入、批量重算、外部 API 调用、高成本模型调用、凭据访问或集群高资源申请。

## 不应触发的场景
- 仅做界面文案调整，不涉及数据或执行风险时。
- 仅做无执行副作用的本地说明文档变更时。
- 仅做普通业务逻辑实现，且不涉及权限、成本、路径、网络或数据破坏风险时。

## 输入
- 数据流说明和字段清单。
- 执行链路与高风险动作清单。
- 既有安全策略与合规约束。
- 目标路径、写入范围、资源申请信息、外部访问需求和回滚条件（如有）。

## 输出
- `risk_level`
- `preflight_checks`
- `human_confirmation_required`
- `dry_run_required`
- `cost_or_quota_impact`
- `circuit_break_conditions`
- `rollback_or_remediation`
- `audit_requirements`
- `decision`
- 在角色专属输出之外，补充统一公共回传头：`role / task_id / scope_in / scope_out / risks / next_actions / ready_for_gate`

## 执行步骤
1. 识别风险操作类型
   - 判断是否涉及文件删除、覆盖写入、批量重算、批量写操作、外部 API、高成本模型、凭据访问、网络访问或高资源调度。
2. 执行前边界检查
   - 检查数据边界、路径边界、权限边界、网络边界、凭据边界和配额边界。
3. 输出前置检查清单
   - 明确必须先完成的检查项，如备份存在、目标目录正确、配额充足、凭据来源合法、dry-run 已完成。
4. 判断是否需要人工确认
   - 对高风险写操作、不可逆操作、高成本调用和大规模任务提交默认要求人工确认。
5. 定义熔断条件
   - 明确在什么条件下停止执行、禁止继续自动化推进。
6. 定义失败后的分流建议
   - 输出 `停止 / 降级 / 重试 / 回滚` 中的推荐路径。
7. 定义审计要求
   - 明确需要记录的路径、资源申请、估计成本、审批动作、恢复动作和结果状态。

## 失败处理
- 若数据边界不清：立即阻断发布并输出整改清单。
- 若策略冲突：按“数据安全优先于自动化效率”原则回退。
- 若审计字段不足：先补全审计规范再允许继续开发。
- 若回滚路径不存在：默认不放行覆盖写或删除操作。
- 若 dry-run 无法完成：默认不放行大规模任务提交。
- 若成本或配额不可估计：默认降级为人工审批。

## 与其他角色的协作方式
- 对 `llm_orchestrator`：
- 以下工具调用必须加 gate：外部 API 调用、高成本模型调用、文件覆盖写、删除、批量写出、凭据读取、网络访问、调度提交。
- 对 `hpc_scheduler`：
- 以下资源申请必须审批：异常高 CPU、异常高内存、长时长、大规模数组任务、全量重算、可能覆盖旧结果的任务。
- 对 `orchestrator`：
- 当发现原始数据删除/覆盖、路径越权、未授权网络访问、回滚不可行、成本失控或审批缺失时，阻断整体任务推进。
- 与 `test_eval`：
- 对齐安全回归、熔断回归、权限边界和 dry-run 校验测试。

## 专门规则
- 禁止直接删除或覆盖原始数据。
- 对批量写操作要求输出影响范围，包括目标目录、文件类型、预计覆盖数量和是否影响旧结果。
- 对高成本模型调用要求输出估计成本或配额影响。
- 对调度系统中的大规模任务要求先 dry-run。
- 对密钥、token、路径越权、网络访问进行显式检查。
- 失败时必须给出“停止 / 降级 / 重试 / 回滚”的分流建议。

## 显式检查项
- `路径检查`：目标路径是否在允许目录内，是否会覆盖现有产物。
- `数据检查`：是否触碰原始数据、是否存在备份、是否可回滚。
- `权限检查`：当前角色是否有权访问目标目录、凭据或服务。
- `网络检查`：外部访问是否被允许、是否需要白名单。
- `成本检查`：模型调用、检索、任务提交是否可能超出预算或配额。
- `调度检查`：是否为高资源、高并发、长作业或全量重算。

## 风险分级表
| 等级 | 典型情形 | 默认动作 |
| --- | --- | --- |
| 低风险 | 只读检查、本地元数据校验、无写入的状态查询 | 允许执行并记录审计摘要 |
| 中风险 | 普通结果目录写入、小范围重试、低成本外部调用 | 通过前置检查后执行，必要时提示确认 |
| 高风险 | 覆盖写入、批量重算、高成本模型调用、跨目录写出、高资源申请 | 默认加 gate，需审计并提供回滚方案 |
| 需要人工批准 | 删除原始数据、覆盖关键输出、访问凭据、未授权网络访问、全量重算且影响历史结果 | 阻断自动执行，必须人工批准 |

## 分流建议模板
- `停止`：存在不可逆破坏、越权访问、回滚不可行或审计缺失。
- `降级`：改为只读模式、样本子集、局部重算、低成本模型或本地替代路径。
- `重试`：仅适用于临时错误、前提检查已通过且不会扩大影响范围。
- `回滚`：恢复旧输出、切回备份、撤销未生效任务或切换到隔离目录。

## 风险审查示例
请求：重建全量基因组预测结果并覆盖旧输出目录。

- 风险等级：
- 高风险，且包含“需要人工批准”因素。
- 前置检查：
- 确认旧输出是否为关键结果目录。
- 确认是否存在可验证备份或快照。
- 确认本次为全量重算还是局部补算。
- 确认目标路径是否在允许目录内。
- 确认 dry-run 是否已生成影响范围清单。
- 是否需要人工确认：
- 是。原因是涉及全量重算与覆盖旧输出。
- 熔断条件：
- 无备份、路径越权、dry-run 缺失、审批缺失、资源申请异常、成本不可控时立即停止。
- 回滚或补救建议：
- 优先改为写入新版本目录而非覆盖旧目录。
- 若必须覆盖，先生成备份清单和回滚说明。
- 若资源过高，先降级为样本子集验证或局部 dry-run。
