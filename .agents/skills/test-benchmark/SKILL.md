name: test-benchmark
description: 面向 test_eval 的测试与 benchmark 技能，用于为 Web/API、LLM workflow、genetics pipeline 和 HPC execution logs 的混合项目建立可重复验证、回归保护与结果记录模板。

## 触发条件
- 新增角色能力、流程节点或关键配置。
- 需要建立发布前质量门槛与评估指标。
- 需要验证安全门禁、熔断链路或调度稳定性。
- 需要为多模块联动系统设计最小可运行测试矩阵。

## 不应触发的场景
- 仅做需求文案润色，不涉及行为变化时。
- 仅做注释与排版修正，不影响执行路径时。
- 已有稳定测试方案，当前只是沿用既有矩阵补跑样例时。

## 输入
- 功能清单与风险优先级。
- 关键流程图与状态机定义。
- 安全边界与调度约束说明。
- API 契约、structured output schema、pipeline 阶段定义和运行日志字段（如有）。

## 输出
- `test_matrix`：测试计划与覆盖矩阵。
- `benchmark_template`：基准案例规格模板。
- `regression_plan`：回归集建议。
- `metrics_and_thresholds`：指标与阈值建议。
- `result_record_format`：结果记录格式。
- `release_blockers`：发布阻断条件。
- 在角色专属输出之外，补充统一公共回传头：`role / task_id / scope_in / scope_out / risks / next_actions / ready_for_gate`

## 执行步骤
1. 划分测试层次
   - 单元测试
   - 集成测试
   - 端到端测试
   - LLM 评测
   - genetics pipeline 评测
   - HPC 日志与执行链路评测
2. 建立覆盖范围
   - Web/API 入口
   - workflow 编排
   - 工具调用接口
   - genetics pipeline 阶段
   - 调度与日志链路
   - 报告生成与结果归档
3. 设计单元测试建议
   - 配置解析
   - 数据校验
   - 任务拆分
   - schema 校验
4. 设计集成测试建议
   - API + workflow + scheduler 接口联动
   - 输入异常时的状态回传
   - 结构化输出与日志字段联动
5. 设计端到端测试建议
   - 从输入请求到报告生成的最小闭环
   - 覆盖成功路径、拒答/降级路径和失败恢复路径
6. 设计 LLM 评测建议
   - 结构化输出正确率
   - 工具调用成功率
   - 拒答/降级行为
   - 输出状态字段与错误字段完整性
7. 设计 genetics pipeline 评测建议
   - 流程完整性
   - 输入异常处理
   - 结果文件完整性
   - 解释边界是否被正确保留
8. 设计 HPC 评测建议
   - 作业提交
   - 阶段衔接
   - 失败重试
   - 日志可追踪性
9. 输出 benchmark 模板与回归计划
   - 统一字段、失败判定和回归比较规则
10. 生成最小可运行测试矩阵
   - 用于当前阶段的 Gate 判断和后续回归扩展。

## 失败处理
- 若可测性不足：回传接口缺口给对应角色并阻断发布。
- 若基准不稳定：先锁定高优先级基准并记录剩余风险。
- 若指标冲突：按安全与正确性优先于性能的原则裁决。
- 若日志字段不足：要求补齐关键状态、错误码和路径索引后再继续。
- 若 schema 不稳定：先冻结结构化输出契约，再建立回归集。
- 若无法构造最小闭环：先退回 orchestrator 缩小测试范围。

## 与其他角色的协作方式
- 与 `orchestrator` 对齐测试里程碑和发布节奏。
- 从 `architect` 获取：
- 模块边界、目录归属、接口契约、状态流和错误语义。
- 从 `llm_orchestrator` 获取：
- structured output schema、tool calling 规则、降级状态、错误字段和上下文摘要字段。
- 从 `popgen_quantgen` 获取：
- QC/结构分析/建模/评估阶段定义、评估指标、输入异常样例、结果文件清单。
- 从 `hpc_scheduler` 获取：
- 作业提交日志、状态转移日志、失败重试日志、dry-run 输出和产物路径索引。
- 与 `safety_fuse`：
- 联合维护安全/熔断回归测试和高风险操作阻断断言。

- `集成测试`

- `端到端测试`

## 最小可运行测试矩阵示例
场景：羊群基因组选择分析智能系统。

- `T1 单元测试`
- 目标：验证配置解析、样本清单校验、schema 输出。
- 输入：最小配置、最小样本映射、最小任务请求。
- 预期：配置可解析、缺字段可报错、schema 字段完整。

- `T2 集成测试`
- 目标：验证 API -> workflow -> scheduler 参数传递。
- 输入：一个最小基因组选择分析请求。
- 预期：生成结构化计划、形成调度请求、记录状态字段。

- `T3 LLM 评测`
- 目标：验证结构化输出和工具调用纪律。
- 输入：正常请求、缺输入请求、高风险请求各一条。
- 预期：正常请求结构化输出完整；缺输入时拒绝猜测；高风险请求触发降级或 gate。

- `T4 genetics pipeline 评测`
- 目标：验证 QC -> 结构分析 -> 建模摘要链路的完整性。
- 输入：最小 VCF/PLINK、表型、协变量占位数据。
- 预期：生成 QC 摘要、结构分析摘要、建模摘要和风险说明索引。

- `T5 HPC 评测`
- 目标：验证作业提交、阶段衔接、失败重试与日志追踪。
- 输入：最小调度请求与模拟失败日志。
- 预期：提交状态可追踪、失败可记录、重试行为符合规则、日志字段完整。
