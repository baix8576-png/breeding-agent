name: popgen-quantgen-pipeline
description: 面向 popgen_quantgen 的流程设计技能，用于规范群体遗传与数量遗传联合分析管线，覆盖输入数据、QC、结构分析、关系矩阵、预测建模、评估和结果解释。

## 触发条件
- 需要定义或评审群体遗传/数量遗传分析流程。
- 需要明确输入合规、流程前置条件、结果产物和解释口径。
- 需要将学科流程转为可编排的工具规范。
- 需要为基因组选择、育种值预测或结构校正建模提供标准路径。

## 不应触发的场景
- 仅做调度队列策略优化，不涉及学科流程时。
- 仅做安全门禁规则，不涉及分析流程时。
- 仅做 prompt 设计、工具路由或检索策略设计时。
- 已有固定流程，仅做资源参数调优时。

## 输入
- 目标分析任务描述。
- 可用工具和环境约束。
- 基因型数据：`VCF` 或 `PLINK`。
- 表型文件。
- 协变量文件。
- 谱系信息（如适用）。
- 物种与群体背景说明（如适用）。

## 输出
- `required_inputs`：标准输入清单。
- `qc_workflow`：原始数据检查与 QC 顺序。
- `structure_analysis`：PCA / 群体结构 / 分层风险分析步骤。
- `relationship_matrix_plan`：亲缘关系或 GRM 构建建议。
- `modeling_plan`：建模候选路径与适用条件。
- `evaluation_plan`：评估指标与验证策略。
- `deliverables`：QC 报告、结构分析摘要、建模摘要、风险说明。
- `exception_checks`：失败处理与异常检查规则。
- 在角色专属输出之外，补充统一公共回传头：`role / task_id / scope_in / scope_out / risks / next_actions / ready_for_gate`

## 执行步骤
1. 明确分析目标与适用范围
   - 说明任务是群体结构分析、关系矩阵构建、基因组选择预测，还是联合流程。
   - 明确适用输入、样本类型、物种背景和限制条件。
2. 原始数据检查
   - 检查文件类型、格式一致性、样本 ID 一致性、位点编码、染色体命名、表型缺失和协变量可用性。
   - 检查谱系信息是否与样本表对齐。
3. 个体与位点质控
   - 个体质控：样本缺失率、杂合度异常、性别/样本标签异常、重复或异常样本。
   - 位点质控：位点缺失率、MAF、HWE、染色体异常位点、编码异常位点。
   - 输出 QC 前后保留数量和过滤理由。
4. 群体结构与分层风险分析
   - 进行 PCA。
   - 进行群体结构分析或等价分层检查。
   - 输出分层风险说明，明确后续建模是否需要纳入主成分或分层策略。
5. 亲缘关系或 GRM 构建
   - 根据任务选择亲缘关系估计、关系矩阵或 `GRM` 构建路径。
   - 标明构建结果用于关系校正、建模输入或异常样本识别。
6. 数量遗传建模建议
   - 至少给出以下候选：
   - `GBLUP`
   - `ssGBLUP`
   - `Bayes` 类方法或等价占位
   - 说明每种路线的适用前提、输入要求和潜在风险。
7. 预测评估建议
   - 相关性评估。
   - 偏差评估。
   - 交叉验证。
   - 分层验证或群体分层验证。
   - 若有谱系或时间分层，建议做独立验证切分。
8. 结果输出模板
   - QC 报告。
   - 结构分析摘要。
   - 建模摘要。
   - 风险说明。
9. 交付给其他角色对齐执行
   - 提供给 llm_orchestrator 做 schema 与 workflow 对齐。
   - 提供给 hpc_scheduler 做资源与阶段映射。
   - 提供给 test_eval 做评估与回归口径定义。

## 失败处理
- 若输入规范缺失：先输出最小输入清单并停止进入建模阶段。
- 若样本 ID、表型或谱系不一致：标记为阻断问题，要求先修复映射关系。
- 若 QC 指标异常：输出异常类型、可能原因和建议复核步骤，不直接进入后续建模。
- 若结构分层显著但模型未校正：标记高风险解释偏差。
- 若工具能力不足：记录能力缺口并请求 orchestrator 排期。
- 若结果解释口径不一致：先统一解释模板，再允许进入报告阶段。

## 异常检查
- 样本缺失率异常升高。
- 位点缺失率异常升高。
- `MAF` 过滤后位点数量骤降。
- `HWE` 异常位点比例过高。
- 杂合度异常样本聚集。
- PCA 显示明显分层或批次效应。
- 关系矩阵出现异常高相似度或重复样本疑点。
- 训练集与验证集存在潜在信息泄漏。

## 与其他角色的协作方式
- 与 `llm_orchestrator` 对齐流程参数接口。
- 与 `hpc_scheduler`：
- 提交资源需求信息，包括样本规模、位点规模、输入文件类型、阶段数量、I/O 密集步骤、内存敏感步骤和预计运行类别。
- 与 `test_eval`：
- 定义评测指标，包括相关性、偏差、交叉验证、分层验证、解释一致性与异常样本识别效果。
- 与 `safety_fuse`：
- 标记高风险数据操作，包括原始基因型导出、表型/谱系明文暴露、完整样本名上云、覆盖过滤后数据、跨目录写出和批量删除中间结果。

## 标准输入
- `genotype_vcf` 或 `genotype_plink`
- `phenotype_table`
- `covariate_table`
- `pedigree_table`（可选）
- `sample_manifest`
- `analysis_config`

## 标准输出
- `qc_report`
- `filtered_dataset_manifest`
- `population_structure_summary`
- `relationship_matrix_manifest`
- `modeling_summary`
- `evaluation_summary`
- `risk_statement`

## 结果输出模板
- `QC 报告`：
- 输入数据概况、过滤阈值、过滤前后样本数/位点数、异常样本/位点说明。
- `结构分析摘要`：
- PCA 概览、群体结构或分层情况、是否建议加入结构校正项。
- `建模摘要`：
- 候选模型、输入配置、训练/验证切分、主要评估指标、适用限制。
- `风险说明`：
- 数据质量风险、分层风险、关系混杂风险、外推风险和解释边界。

## scripts/ 规划
- `scripts/qc_pipeline/`
- 预留个体与位点质控脚本、阈值配置模板和过滤日志汇总脚本。
- `scripts/pca_pipeline/`
- 预留 PCA、结构分析、批次/分层摘要脚本。
- `scripts/grm_builder/`
- 预留亲缘关系估计、GRM 构建和矩阵摘要脚本。
- `scripts/genomic_prediction/`
- 预留 GBLUP、ssGBLUP、Bayes 或等价建模流程脚本。
- `scripts/report_generator/`
- 预留 QC 报告、结构摘要、建模摘要和风险说明生成脚本。

## references/ 规划
- `references/input-specs/`
- 存放 VCF、PLINK、表型、协变量、谱系文件规范。
- `references/qc-rules/`
- 存放缺失率、MAF、HWE、杂合度、异常样本识别规则说明。
- `references/structure-analysis/`
- 存放 PCA、群体结构、分层风险解释模板。
- `references/modeling-guides/`
- 存放 GBLUP、ssGBLUP、Bayes 或等价方法的适用场景说明。
- `references/evaluation/`
- 存放相关性、偏差、交叉验证、分层验证的评估口径。
- `references/report-templates/`
- 存放 QC 报告、结构分析摘要、建模摘要、风险说明模板。

## 示例：羊群基因组选择分析项目
- 输入：
- 羊群基因型数据（VCF 或 PLINK）、表型文件、协变量文件、谱系信息、样本映射表。
- 流程阶段：
- 原始数据检查 -> 个体与位点 QC -> PCA/群体结构分析 -> 亲缘关系或 GRM 构建 -> GBLUP/ssGBLUP/Bayes 候选建模 -> 交叉验证与分层验证 -> 报告输出。
- 输出清单：
- QC 报告、过滤后数据清单、结构分析摘要、关系矩阵索引、建模摘要、评估摘要、风险说明。
